# UIにやってほしいこと

## 前提

- APIは実装済み・動作確認済み。詳細仕様は [API.md](API.md) を参照（エンドポイント一覧・リクエスト/レスポンス例）
- 開発中のBase URL: `http://localhost:3000`（`npm start` で起動。CORS許可済みなのでブラウザから直接叩ける）
- マスタデータ（商品48件・ジャンル9種）は投入済みなので、モック無しで実データに近い状態で開発できる

## 作ってほしい画面（企画時点のたたき台）

1. **商品一覧画面**
   - `GET /api/products` を使用。ジャンル絞り込みは `?genre_id=` に対応済み
   - 各商品に `current_price`（現行売値）・`current_stock`（最新在庫数）が含まれる
2. **在庫数・仕入数の入力画面**
   - 在庫記録: `POST /api/stock-records`（同日の再送信は上書きされる）
   - 仕入れ記録: `POST /api/purchases`
3. **予測結果・おすすめ発注数の表示画面**
   - **現時点で未実装**。AI班の推奨購買数量をAPIで返す仕組みがまだ無いため、ここは後回しにして①②から着手してほしい（[docs/ai.md](ai.md) 参照）

## お願いしたいこと

- 上記1・2から実装を始めてほしい（3はバックエンド側のAPI追加待ち）
- API側で足りないデータ・欲しいレスポンス形式があれば教えてほしい（`src/routes/` を直接改修してもらっても、要望を伝えてもらってこちらで直してもOK）
- 商品登録・編集画面が必要か（現状は企画上「商品はDB班のマスタで固定」を想定しているが、運用上追加が必要なら `POST/PUT /api/products` は用意済み）
